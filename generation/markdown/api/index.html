    API Reference¶

    This reference provides comprehensive documentation of FynX's public API. Whether you're building your first reactive application or exploring advanced patterns, you'll find detailed explanations, type signatures, and examples for every class, decorator, and operator FynX provides.

    How to Navigate This Reference¶

    The API documentation is organized by functional area to help you find what you need quickly. If you're new to FynX, start with the Core API and Observable pages to understand the fundamentals. If you're looking for a specific feature, use the sections below to jump directly to the relevant documentation.

    Core API¶

    Core API - Start here for an overview of FynX's main exports and public interface. This page documents the primary functions and classes you'll import and use in your applications, along with guidance on which APIs to use for common tasks.

    Observable Types¶

    Observables are the building blocks of reactive systems in FynX. Each type serves a specific purpose in your reactive architecture:

    • Observable - The foundational reactive value type. Learn how to create, read, update, and subscribe to basic observables. This is the most commonly used type and the basis for understanding all other observable variants.

    • ComputedObservable - Derived values that automatically recompute when their dependencies change. Understand how computed observables provide memoization, lazy evaluation, and efficient reactive transformations.

    • MergedObservable - Observables that combine multiple sources into a single reactive value. Discover how to aggregate state from different observables and react to changes in any of them.

    • ConditionalObservable - Observables that only emit when specific conditions are satisfied. Learn how to implement conditional logic, filtering, and state machine patterns in your reactive code.

    • Observable Descriptors - The mechanism that makes class attributes reactive. Explore how descriptors enable the convenient observable("value") syntax in Store classes and how they integrate with Python's data model.

    • Observable Operators - Composable operators for transforming and combining observables. Master the |, >>, &, and ~ operators to build expressive reactive pipelines with clear data flow.

    Reactive Decorators and Utilities¶

    Decorators provide the high-level API for creating reactive relationships in your application:

    • @computed - Transform methods into computed properties that automatically track dependencies. Learn how to create efficient derived state with automatic memoization and minimal recalculation.

    • @reactive - Execute functions automatically when observables change. Understand how to implement side effects, UI updates, and event handling in response to state changes.

    • @watch - Run functions only when specific conditions are met. Discover how to implement conditional reactions, validation triggers, and state machine transitions declaratively.

    • Store & @observable - Organize related state into cohesive units. Learn how to create Store classes, make attributes reactive with the @observable decorator, and manage complex application state with clear boundaries.

    API at a Glance¶

    For quick lookup, here's a categorized overview of FynX's API surface:

    Reactive State¶

    These are the core types you'll use to create and manage reactive state:

    • Observable - Mutable reactive values with automatic change notification
    • Store - Containers that group related observables and computed values
    • ReactiveContext - The execution context that tracks dependencies and manages updates

    Derived Values¶

    Transform and combine observables to create new reactive values:

    • >> operator - Create derived values with automatic dependency tracking
    • ComputedObservable - The underlying computed value type (usually created via >> operator)

    Reactive Effects¶

    Execute code in response to observable changes:

    • @reactive - Decorator for functions that run when dependencies change
    • @watch - Decorator for conditional reactive functions
    • subscribe() - Method for programmatic subscription to observable changes

    Composition¶

    Combine and transform observables to build complex reactive relationships:

    • | operator - Merge multiple observables into tuple values
    • & operator - Filter observables based on conditions
    • ~ operator - Negate boolean observables
    • MergedObservable - Combine multiple observable sources
    • ConditionalObservable - Apply conditional logic to reactive values

    Class Integration¶

    Make class attributes reactive:

    • @observable - Decorator that makes class attributes reactive (for Store classes)
    • observable("value") - Class-level declaration syntax for reactive attributes
    • Observable descriptors - The mechanism enabling reactive class attributes

    Getting Started¶

    Here's a complete example demonstrating the key APIs working together:

    from datetime import datetime
from fynx import Store, observable, reactive, watch

# Store: organize related reactive state
class UserStore(Store):
    # observable: declare reactive class attributes
    name = observable("Alice")
    age = observable(30)
    is_adult = observable(False)

    # >> operator: derive values that auto-update
    greeting = (name | age) >> (
        lambda n, a: f"Hello, {n}! You are {a} years old."
    )

    # >> operator: another derived value
    birth_year = age >> (lambda a: datetime.now().year - a)

# @reactive: run side effects when dependencies change
@reactive(UserStore.name, UserStore.age)
def log_user_changes(name, age):
    print(f"User updated: {name}, {age} years old")

# @watch: run effects only when conditions are met
@watch(
    lambda: UserStore.age.value >= 18,
    lambda: UserStore.is_adult.value
)
def on_verified_adult():
    print(f"{UserStore.name.value} is a verified adult")

# Operators: compose observables functionally
adult_and_online = UserStore.is_adult & UserStore.is_online

# All changes propagate automatically
UserStore.name = "Bob"        # Triggers: log_user_changes
UserStore.age = 25            # Triggers: log_user_changes, greeting recalc
UserStore.is_adult = True     # Triggers: on_verified_adult (if age >= 18)

    • Notes highlight important behaviors, edge cases, or performance considerations
    • See also links connect related concepts and alternative approaches
